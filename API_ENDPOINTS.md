# Документация и Архитектура API

В данном файле описана логика работы системы, её ключевые компоненты и подробный список API эндпоинтов.

---

## Обзор системы
Проект представляет собой платформу для управления виртуальными ресурсами (IaaS). Основная задача системы — позволить пользователям создавать проекты (тенанты), приобретать для них квоты ресурсов через тарифные планы и запускать виртуальные машины в рамках этих квот.

### Ключевые сущности
1.  **Пользователи (Users)**: Основные участники системы. Имеют роли (`USER`, `ADMIN`).
2.  **Тенанты (Tenants)**: Изолированные рабочие пространства. Каждый тенант имеет владельца, но в него могут быть добавлены и другие пользователи.
3.  **Планы (Plans)**: Шаблоны ресурсов. Описывают лимиты (CPU, RAM, Disk) и стоимость.
4.  **Квоты тенантов (Tenant Quotas)**: Экземпляр купленного плана для конкретного тенанта. Именно оплаченная квота ("PAID") открывает возможность создания ВМ.
5.  **Виртуальные машины (Virtual Machines)**: Непосредственные вычислительные ресурсы, потребляющие лимиты из квоты тенанта.
6.  **Логи ВМ (VM Logs)**: История событий и действий, происходящих с виртуальными машинами.
7.  **Заявки на кастомные планы (Custom Plan Requests)**: Механизм переговоров между пользователем и админом для получения тарифа с индивидуальными характеристиками.

---

## Как это работает (Бизнес-логика)

### 1. Аутентификация и Доступ
Система использует **JWT (JSON Web Tokens)**. После успешного входа через `/auth/login` пользователь получает токен, который необходимо передавать в заголовке `Authorization: Bearer <token>` для защищенных запросов.

### 2. Жизненный цикл ресурсов
Процесс запуска ВМ выглядит следующим образом:
1.  Админ создает тарифные **Планы** (`/plans`).
2.  Пользователь создает **Тенант** (`/tenants`).
3.  Тенанту назначается **Квота** на основе выбранного плана (`/tenant_quotas`). 
4.  Как только статус оплаты квоты становится `PAID`, тенант получает доступный пул ресурсов.
5.  Пользователь создает **ВМ** (`/virtual_machines`).

### 3. Валидация квот (Resource Guards)
При создании или обновлении любой ВМ система автоматически:
*   Считает общую сумму всех ресурсов (CPU, RAM, Disk) по всем квотам тенанта со статусом `PAID`.
*   Считает текущее потребление ресурсов всеми запущенными ВМ в этом тенанте.
*   Блокирует операцию (400 Bad Request), если новые параметры ВМ приведут к превышению суммарного оплаченного лимита.

## Список Эндпоинтов

### Auth (`/auth`)
*   **POST** `/auth/login` - Вход и получение токена.

### Users (`/users`)
*   **GET** `/users/` - Список всех пользователей.
*   **GET** `/users/roles` - Справочник ролей (`USER`, `ADMIN`).
*   **GET** `/users/{identifier}` - Просмотр профиля по ID или логину.
*   **POST** `/users/` - Регистрация.
*   **PUT** `/users/{user_id}` - Обновление данных.
*   **DELETE** `/users/{user_id}` - Удаление.

### Plans (`/plans`)
*   **GET** `/plans/` - Список тарифов.
*   **GET** `/plans/{plan_id}` - Детали тарифа.
*   **POST** `/plans/` - Создание плана (Admin).
*   **PUT** `/plans/{plan_id}` - Изменение плана.
*   **DELETE** `/plans/{plan_id}` - Удаление.

### Tenants (`/tenants`)
*   **GET** `/tenants/` - Все тенанты.
*   **GET** `/tenants/{tenant_id}` - Детали тенанта.
*   **GET** `/tenants/owner/{owner_id}` - Тенанты конкретного владельца.
*   **POST** `/tenants/` - Создание тенанта.
*   **PUT** `/tenants/{tenant_id}` - Изменение названия или статуса.
*   **DELETE** `/tenants/{tenant_id}` - Удаление.
*   **GET** `/tenants/{tenant_id}/users` - Список участников тенанта.

### Tenant Quotas (`/tenant_quotas`)
*   **GET** `/tenant_quotas/` - Список квот.
*   **GET** `/tenant_quotas/payment_statuses` - Статусы оплаты (`UNPAID`, `PAID`, `REFUNDED`).
*   **GET** `/tenant_quotas/tenant/{tenant_id}` - Квоты проекта.
*   **POST** `/tenant_quotas/` - Выставление счета/назначение квоты.
*   **PUT** `/tenant_quotas/{quota_id}` - Обновление (напр. смена статуса на `PAID`).
*   **DELETE** `/tenant_quotas/{quota_id}` - Удаление.

### Virtual Machines (`/virtual_machines`)
*   **GET** `/virtual_machines/` - Список ВМ (фильтры по тенанту/создателю).
*   **GET** `/virtual_machines/statuses` - Статусы ВМ (`ACTIVE`, `STOPPED`, `INACTIVE`).
*   **GET** `/virtual_machines/{vm_id}` - Детали ВМ.
*   **POST** `/virtual_machines/` - Создание ВМ (**с проверкой квоты**).
*   **PUT** `/virtual_machines/{vm_id}` - Изменение конфигурации (**с проверкой квоты**).
*   **DELETE** `/virtual_machines/{vm_id}` - Удаление.

### VM Logs (`/vm_logs`)
*   **GET** `/vm_logs/` - Все логи системы.
*   **GET** `/vm_logs/vm/{vm_id}` - События конкретной машины.
*   **POST** `/vm_logs/` - Добавление события.

### Custom Plan Requests (`/custom_plan_requests`)
*   **GET** `/custom_plan_requests/` - Список заявок.
*   **GET** `/custom_plan_requests/statuses` - Статусы заявок.
*   **POST** `/custom_plan_requests/` - Создание заявки от пользователя.
*   **PUT** `/custom_plan_requests/{request_id}` - Рассмотрение заявки админом.
